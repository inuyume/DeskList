# DeskList 开发设计文档

## 1. 项目概述

### 1.1 项目名称

暂定名称：**DeskList**

其他可选名称：

- TodoPin
- FloatList
- TaskDock
- CornerTodo
- 桌面清单

### 1.2 项目定位

DeskList 是一款面向 Windows 的轻量桌面待办工具。

应用以小窗口形式常驻桌面，支持创建多个任务列表、快速添加任务、勾选已完成任务，并提供始终置顶、系统托盘、全局快捷键和开机启动功能。

DeskList 不定位为复杂的项目管理平台，不提供团队协作、云端账号、富文本编辑和复杂工作流。

### 1.3 核心价值

用户可以：

1. 随时创建新的 List。
2. 在当前 List 中快速添加任务。
3. 单击任务前的圆形复选框完成任务。
4. 使用快捷键快速显示或隐藏应用。
5. 将应用固定在桌面最前方。
6. 在不登录账号、不连接服务器的情况下使用。
7. 关闭并重新打开应用后保留全部数据。

## 2. 项目目标

### 2.1 产品目标

构建一个：

- 启动速度快
- 空闲 CPU 占用接近零
- 无后台服务器
- 无持续网络请求
- 数据完全保存在本地
- 操作路径短
- 适合长期驻留桌面

的 Windows 待办应用。

### 2.2 性能目标

以下内容作为验收目标，而不是绝对保证值：

- 冷启动尽量控制在 1 秒左右。
- 空闲 CPU 占用接近 0%。
- 不存在轮询任务。
- 不存在后台 HTTP 服务。
- 页面只使用一个 WebView 窗口。
- 正式安装包尽量保持较小。
- 任务数量达到 5000 条时仍可正常使用。
- 普通增删改操作应立即响应。
- 窗口隐藏时不执行动画和定时刷新。

### 2.3 平台范围

第一版只支持：

- Windows 10 64 位
- Windows 11 64 位

暂不要求：

- macOS
- Linux
- Android
- iOS
- Web 版本

代码结构应尽量避免硬编码 Windows 路径，为未来跨平台保留可能性。

## 3. 技术选型

### 3.1 技术栈

采用：

- Tauri 2
- Vue 3
- TypeScript
- Vite
- Composition API
- 原生 CSS
- Tauri Store 插件
- Tauri Global Shortcut 插件
- Tauri Autostart 插件
- Tauri Window State 插件
- Tauri 系统托盘 API

不使用：

- Electron
- Node.js 后端
- Express
- PostgreSQL
- MySQL
- Redis
- Docker
- 完整 UI 组件库
- 远程 API
- 用户登录系统

### 3.2 前端状态管理

第一版不引入 Pinia。

使用一个集中式 composable 管理状态：

```text
useTodoStore.ts
```

它负责：

- Lists 状态
- Tasks 状态
- 当前选中的 List
- 设置状态
- 数据增删改
- 持久化调度
- 数据校验

当项目规模明显扩大时再考虑 Pinia。

### 3.3 本地存储

第一版使用 Tauri Store 插件进行持久化。

逻辑文件名：

```text
desklist.json
```

数据应写入 Tauri 应用数据目录，不允许写到项目目录或程序安装目录。

必须满足：

- 数据变更后自动保存。
- 保存失败时给出非阻塞错误提示。
- 启动时读取已有数据。
- 文件不存在时创建默认数据。
- 数据损坏时尝试恢复。
- 保留最近一次有效备份。
- 不允许因为单条异常任务导致整个应用无法启动。

### 3.4 不使用 SQLite 的原因

第一版的数据关系简单，主要是 List 与 Task 两层结构。

本项目预计只有：

- 少量 List
- 数百至数千条 Task
- 简单本地查询
- 无并发写入
- 无复杂统计

因此无需引入 SQLite。

以下情况出现后再评估 SQLite：

- 任务数量达到数万条。
- 增加复杂标签和组合筛选。
- 增加全文搜索。
- 增加历史操作日志。
- 增加数据统计。
- 增加多设备同步。

## 4. 功能范围

### 4.1 List 管理

必须实现：

- 创建 List
- 切换 List
- 重命名 List
- 删除 List
- List 排序
- 显示每个 List 的未完成任务数量
- 保存上次打开的 List
- 至少保留一个默认 List

默认 List：

```text
今天
```

删除 List 时：

1. 必须弹出确认对话框。
2. 明确提示其中任务也会被删除。
3. 如果只剩一个 List，不允许直接删除。
4. 删除当前 List 后自动切换到相邻 List。

#### List 名称规则

- 去除首尾空格。
- 不允许为空。
- 最大长度 40 个字符。
- 可以重名，但界面应允许用户区分。
- 创建后立即进入该 List。

### 4.2 Task 管理

必须实现：

- 添加任务
- 勾选完成
- 取消完成
- 编辑任务标题
- 删除任务
- 调整任务顺序
- 显示或隐藏已完成任务
- 清除当前 List 的已完成任务
- 将已完成任务排在未完成任务之后

#### 添加任务规则

- 输入框始终位于任务列表底部。
- 按 Enter 添加任务。
- 输入为空时不添加。
- 自动去除首尾空格。
- 最大长度 300 个字符。
- 新任务添加到当前 List。
- 添加成功后清空输入框。
- 添加成功后输入框保持焦点。

#### 编辑任务规则

- 双击任务标题进入编辑状态。
- Enter 保存。
- Escape 取消。
- 失去焦点时保存。
- 编辑为空时恢复原内容，不自动删除任务。

#### 完成任务规则

- 点击任务前的复选按钮切换完成状态。
- 完成后文字显示删除线或弱化效果。
- 已完成任务移动到任务列表底部。
- 保存 `completedAt`。
- 取消完成时清除 `completedAt`。

### 4.3 窗口功能

必须实现：

- 始终置顶开关
- 记住窗口位置
- 记住窗口大小
- 窗口最小尺寸
- 关闭按钮隐藏到托盘
- 托盘菜单真正退出
- 单击托盘图标显示或隐藏窗口
- 全局快捷键显示或隐藏窗口
- 窗口显示时自动聚焦快速添加输入框

建议窗口配置：

```text
默认宽度：440
默认高度：560
最小宽度：340
最小高度：400
默认居中：是
默认始终置顶：否
允许最大化：否
允许调整大小：是
```

第一版保留系统原生标题栏。

不要在第一版实现：

- 无边框窗口
- 透明窗口
- 亚克力模糊
- 毛玻璃效果
- 桌面吸附动画
- 多窗口

### 4.4 系统托盘

托盘菜单包含：

```text
显示 DeskList
快速添加
始终置顶
开机启动
退出
```

行为要求：

- 点击窗口关闭按钮时隐藏窗口，不退出进程。
- 选择“退出”时才真正结束进程。
- 第一次关闭到托盘时，可以显示一次提示。
- 后续不重复提示。
- 托盘图标双击时也可以显示主窗口。

### 4.5 全局快捷键

默认快捷键：

```text
Ctrl + Alt + Space
```

功能：

- 应用隐藏时显示窗口。
- 应用显示时隐藏窗口。
- 显示窗口后置于前台。
- 自动聚焦任务输入框。

要求：

- 快捷键注册失败不能导致程序崩溃。
- 注册失败时在设置页面显示提示。
- 后续可以允许用户修改快捷键。
- 第一版可以只使用固定快捷键。

不要默认使用：

```text
Alt + Space
```

因为它与 Windows 系统窗口菜单存在冲突。

### 4.6 开机启动

设置页面提供：

```text
开机启动
```

默认关闭。

开启后：

- 用户登录 Windows 时启动 DeskList。
- 建议启动后默认隐藏到托盘。
- 不主动抢占当前窗口焦点。

关闭后：

- 移除自动启动配置。

### 4.7 设置页面

第一版设置项：

- 始终置顶
- 开机启动
- 启动后隐藏到托盘
- 显示已完成任务
- 完成任务自动移到底部
- 全局快捷键状态
- 导出数据
- 导入数据
- 重置应用数据
- 关于 DeskList

设置页面使用主窗口内视图，不创建独立窗口。

## 6. 数据模型

### 6.1 根数据结构

```ts
export interface AppData {
  version: number;
  activeListId: string;
  lists: TodoList[];
  settings: AppSettings;
}
```

### 6.2 List 模型

```ts
export interface TodoList {
  id: string;
  name: string;
  sortOrder: number;
  createdAt: string;
  updatedAt: string;
  tasks: TodoTask[];
}
```

### 6.3 Task 模型

```ts
export interface TodoTask {
  id: string;
  title: string;
  completed: boolean;
  sortOrder: number;
  createdAt: string;
  updatedAt: string;
  completedAt: string | null;
}
```

### 6.4 Settings 模型

```ts
export interface AppSettings {
  alwaysOnTop: boolean;
  launchAtStartup: boolean;
  startHidden: boolean;
  showCompleted: boolean;
  moveCompletedToBottom: boolean;
  closeToTrayHintShown: boolean;
}
```

### 6.5 默认数据

```ts
export const DEFAULT_APP_DATA: AppData = {
  version: 1,
  activeListId: "default-list",
  lists: [
    {
      id: "default-list",
      name: "今天",
      sortOrder: 0,
      createdAt: new Date().toISOString(),
      updatedAt: new Date().toISOString(),
      tasks: []
    }
  ],
  settings: {
    alwaysOnTop: false,
    launchAtStartup: false,
    startHidden: false,
    showCompleted: true,
    moveCompletedToBottom: true,
    closeToTrayHintShown: false
  }
};
```

ID 优先使用：

```ts
crypto.randomUUID()
```

不要使用数组索引作为 ID。

时间统一保存为 ISO 8601 UTC 字符串。

## 7. 数据持久化设计

### 7.1 保存策略

以下操作后立即保存：

- 添加任务
- 删除任务
- 勾选任务
- 创建 List
- 删除 List
- 修改设置

以下操作使用 300 毫秒防抖保存：

- 编辑任务标题
- 重命名 List
- 拖拽排序
- 调整窗口相关设置

保存流程：

```text
修改内存状态
→ 更新 updatedAt
→ 校验数据
→ 调用持久化服务
→ 保存成功
```

失败流程：

```text
保存失败
→ 保留内存状态
→ 显示错误提示
→ 写入应用日志
→ 允许用户再次尝试
```

### 7.2 备份策略

每次覆盖主要数据前：

1. 读取当前已保存数据。
2. 将当前有效数据保存为备份。
3. 再写入最新数据。

只保留一个最近备份即可。

逻辑名称：

```text
desklist.json
desklist.backup.json
```

### 7.3 启动恢复策略

启动时依次执行：

1. 加载主要数据。
2. 校验 `version`、`lists` 和 `settings`。
3. 如果主要数据有效，正常启动。
4. 如果主要数据无效，尝试加载备份。
5. 如果备份有效，恢复备份并提示用户。
6. 如果两者均不可用，加载默认数据。
7. 不允许应用白屏或无限加载。

### 7.4 导入与导出

导出：

- 导出完整 `AppData`。
- 文件扩展名为 `.json`。
- 文件名包含日期。
- 示例：`desklist-backup-2026-07-15.json`。

导入：

- 用户选择 JSON 文件。
- 先解析和校验。
- 显示导入确认。
- 导入前备份当前数据。
- 导入失败不得破坏当前数据。
- 导入成功后刷新界面。

## 8. UI 与交互设计

### 8.1 主界面布局

桌面宽度充足时使用双栏布局：

```text
┌──────────────────────────────────────┐
│ DeskList                       设置  │
├─────────────┬────────────────────────┤
│ 今天      3 │ 今天                   │
│ Homelab   2 │                        │
│ 学习      5 │ ○ 配置 Prometheus      │
│ 求职      1 │ ○ 整理 K3s 文档        │
│             │ ● 安装 Cilium          │
│ + 新建列表  │                        │
│             │ + 添加任务             │
└─────────────┴────────────────────────┘
```

窗口较窄时：

- 侧边栏可折叠。
- 顶部显示当前 List 名称。
- 通过菜单按钮打开 List 抽屉。
- 不允许出现横向滚动条。

### 8.2 视觉风格

风格要求：

- 浅色简洁
- 不使用大面积渐变
- 不使用复杂阴影
- 不使用持续动画
- 不使用背景视频
- 不使用玻璃拟态
- 保持较高信息密度
- 控件圆角适中
- 文字清晰
- 复选框点击区域足够大

建议设计变量：

```css
:root {
  --background: #f7f7f8;
  --surface: #ffffff;
  --surface-hover: #f1f1f2;
  --text-primary: #1f2328;
  --text-secondary: #6e7781;
  --border: #d8dee4;
  --accent: #2563eb;
  --danger: #dc2626;
  --radius-small: 6px;
  --radius-medium: 10px;
}
```

### 8.3 任务项交互

每个任务项包含：

- 完成复选按钮
- 任务标题
- 删除按钮
- 可选拖拽区域

删除按钮：

- 默认弱化。
- 鼠标悬停任务时显示。
- 删除任务不要求二次确认。
- 删除 List 必须确认。

任务完成动效：

- 只允许短暂的 CSS transition。
- 持续时间不超过 150ms。
- 不引入动画库。

### 8.4 空状态

当前 List 没有任务时显示：

```text
这里还没有任务

在下方输入内容并按 Enter 添加。
```

### 8.5 错误提示

采用轻量 Toast：

- 保存失败
- 导入失败
- 快捷键注册失败
- 数据恢复成功
- 操作不允许

Toast 自动消失时间：

```text
3～5 秒
```

错误信息必须对用户可理解，技术细节写入日志。

## 10. Tauri 集成设计

### 10.1 插件

使用官方插件：

- Store
- Global Shortcut
- Autostart
- Window State
- Dialog
- File System

只安装实际使用的插件。

### 10.2 权限原则

Tauri capabilities 遵循最小权限原则。

只开放：

- 当前应用需要的 Store 权限
- 必要文件读写权限
- 对话框权限
- 全局快捷键权限
- 开机启动权限
- 窗口显示、隐藏和置顶权限

不要开放：

- 任意 Shell 命令
- 任意外部程序执行
- 无限制文件系统访问
- 任意 HTTP 请求
- 不需要的剪贴板权限

### 10.3 窗口关闭逻辑

监听主窗口关闭事件：

```text
用户点击关闭
→ 阻止默认退出
→ 隐藏主窗口
```

真正退出时：

```text
托盘菜单点击退出
→ 设置退出标志
→ 保存数据
→ 注销全局快捷键
→ 退出应用
```

必须避免“退出菜单也只隐藏窗口”的死循环。

### 10.4 单实例

建议启用单实例行为。

再次启动应用时：

- 不创建第二个主窗口。
- 显示已有窗口。
- 将已有窗口置于前台。
- 聚焦快速添加输入框。

## 11. 轻量化约束

开发时必须遵守：

1. 不使用 Electron。
2. 不运行本地 Web 服务器作为正式架构。
3. 不引入完整组件库。
4. 不引入动画库。
5. 不引入日期处理库，使用原生 `Date`。
6. 不引入 Axios，因为项目无远程请求。
7. 不引入 Lodash。
8. 不引入大型图标包的完整集合。
9. 图标按需使用，或使用少量内联 SVG。
10. 只创建一个主窗口。
11. 不使用持续定时器。
12. 不执行后台轮询。
13. 不进行自动网络访问。
14. 不保存无意义日志。
15. 不在前端状态中保存重复数据。
16. 不为简单 CRUD 编写复杂抽象层。
17. 不使用虚拟列表库，除非性能测试证明必要。
18. 正式版本关闭开发工具。
19. 正式版本使用 release 构建。
20. 所有依赖都要说明用途。

## 12. 异常处理

必须处理：

- 数据文件不存在
- 数据文件为空
- JSON 无法解析
- 数据版本未知
- `activeListId` 指向不存在的 List
- `lists` 为空
- Task 缺少 ID
- 重复 ID
- 保存失败
- 快捷键被占用
- 开机启动设置失败
- 导入文件格式错误
- 用户取消文件选择
- 窗口位置位于已断开的显示器之外

恢复规则：

- `activeListId` 无效时选择第一个 List。
- `lists` 为空时创建默认 List。
- 非法任务尽量跳过，不阻止其他数据加载。
- 窗口不可见时恢复到主显示器中央。
- 不允许使用 `any` 逃避类型检查。
- 不允许吞掉异常而没有日志或用户提示。
